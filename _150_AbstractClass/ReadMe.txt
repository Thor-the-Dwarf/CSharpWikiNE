"Abstrakt" bedeutet, dass etwas nicht vollständig definiert ist.
Abstrakte Klassen in C# sind Klassen, die als 'abstract' gekennzeichnet sind. 
Sie können eine Mischung aus abstrakten und konkreten Membern enthalten:
- Abstrakte Member (Methoden, Eigenschaften) sind ohne Implementierung und müssen in abgeleiteten Klassen definiert werden.
- Konkrete Member haben eine vollständige Implementierung, die von abgeleiteten Klassen verwendet oder überschrieben werden kann.
Direkte Instanzen von abstrakten Klassen können nicht erstellt werden. 
Abstrakte Klassen dienen als Basis für andere Klassen und helfen dabei, eine gemeinsame Struktur zu definieren.
Der Hauptzweck abstrakter Klassen besteht darin, dass sie von anderen Klassen erweitert werden. 
Dies fördert das Prinzip der Wiederverwendung von Code und Polymorphismus.
